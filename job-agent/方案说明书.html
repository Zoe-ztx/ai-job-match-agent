「职引」AI求职智能匹配智能体

方案说明书 · 2026年6月

一、问题诊断

当前大学生求职面临三大核心痛点：

  • 信息不对称：求职者难以全面了解自身背景匹配的岗位，海投效率低下，简历初筛命中率不足20%；
  • 简历质量参差不齐：多数学生缺乏简历撰写经验，无法准确突出自身优势与岗位需求的匹配点；
  • 职业规划迷茫：不清楚同背景（同学历、同专业）求职者的去向分布，缺乏决策参考数据。

传统解决方案依赖人工咨询或通用招聘平台，存在成本高、响应慢、个性化程度低等问题。基于以上痛点，本项目旨在利用AI大模型能力，打造一款面向高校学生的智能求职助手。

二、方案设计

2.1 整体架构

采用前后端分离架构：前端为单页面应用（SPA），后端基于Node.js + Express框架，通过API调用腾讯混元大模型实现智能化功能。整体分为三层——展示层（HTML/CSS/JS）、服务层（Node.js API）、AI层（腾讯混元大模型）。

2.2 核心功能模块

功能模块 核心能力 用户价值
岗位智能匹配 解析用户背景 → AI检索匹配岗位 → 结构化卡片展示 提升初筛命中，减少无效投递
简历优化诊断 上传简历 → AI逐项评分 → 问题定位+优化建议 量化简历质量，提供可执行改进方案
同背景去向分析 分析用户背景 → 生成城市/薪资/岗位去向报告 为城市选择和薪资谈判提供参考

2.3 交互流程设计

采用三屏式交互：首页展示功能卡片 → 简历上传页（支持PDF/DOC/DOCX/TXT）→ AI对话页。系统内置降级机制：当后端服务不可用时，自动回退至本地关键词匹配模式，保证基础可用性。

UI设计以紫色(#6A1B9A)和金色(#FFD700)为主色调，搭配白色背景，并添加浮动emoji贴纸、装饰圆点等可爱元素增强趣味性。各功能加载时显示动态倒计时预估（60秒），降低用户等待焦虑。

三、AI工具选型理由

本项目的AI能力由腾讯混元大模型（hunyuan-turbos-latest）提供支撑，选型理由如下：

  • 中文理解能力强：混元模型在中文语义理解、长文本生成方面表现优异，适合处理简历解析、岗位匹配等中文场景；
  • API调用便捷：提供标准的OpenAI兼容接口，集成成本低，支持流式输出提升用户体验；
  • 成本可控：按token计费模式适合学生项目预算，turbo版本在速度和成本间取得良好平衡；
  • 生态完善：官方SDK支持Node.js，与本项目技术栈无缝衔接。

对比其他方案：GPT-4o效果优秀但国内访问受限且成本较高；通义千问和文心一言也是可选方案，但混元在本项目中已验证稳定性和响应速度满足需求。

四、关键配置

4.1 技术栈

  • 前端：原生HTML5 + CSS3 + JavaScript（单文件架构，无构建依赖）；
  • 后端：Node.js 22 + Express 4.x + Multer（文件上传）；
  • 文件解析：pdf-parse（PDF）、mammoth（DOCX）；
  • AI接口：腾讯混元 hunyuan-turbos-latest。

4.2 API端点设计

端点 说明
/api/upload-resume简历文件上传与文本提取
/api/parse-resumeLLM解析简历为结构化信息
/api/chat主对话接口（支持流式输出）
/api/market-analysis同背景去向分析
/api/set-context用户上下文信息设置
/api/health服务健康检查

4.3 提示词工程

针对每个功能设计了专用System Prompt和模板化User Prompt。其中同背景去向分析使用独立模块（background-analysis-prompt.js）维护提示词模板，确保输出包含城市分布、薪资区间、岗位情况等结构化信息，并标注数据来源可信度等级。

六、效果评估

6.1 功能完整性

三大核心功能均已实现并可独立运行：岗位匹配输出含匹配度标签的结构化岗位卡片；简历诊断提供评分（满分100）、问题分类和具体修改建议；同背景去向分析输出城市分布热力图、薪资区间表和就业方向建议。

6.2 用户体验

三屏交互流程清晰，从功能选择到结果展示不超过3次点击；60秒倒计时有效管理等待预期；紫色/金色主题配合可爱贴纸元素获得正面视觉反馈；支持PDF/DOC/DOCX/TXT四种主流简历格式。

6.3 技术指标

  • 代码量约1900行（含注释）；
  • 7次Git提交记录完整迭代过程；
  • 8个RESTful API端点；
  • 内置降级机制保障离线基础可用性；
  • 响应式布局适配移动端访问。

6.4 待改进方向

  • 可接入真实招聘数据源替代AI生成的模拟数据；
  • 可增加用户登录和历史记录持久化；
  • 可扩展更多文件格式支持（如RTF）；
  • 演示视频可作为补充材料进一步展示交互流程。
